System Architecture

Full-stack blueprint — frontend, backend, data pipeline, queues, and on-chain layer.

🖥

Frontend

Next.js 14 App Router with server components. Real-time updates via WebSocket. Tailwind + shadcn/ui for design system.

Next.js 14 TypeScript Tailwind shadcn/ui WebSocket NextAuth.js
⚙

Backend API

Node.js + Fastify for high-throughput REST API. Handles OAuth exchange, job dispatch, signal queries. JWT session auth.

Node.js Fastify TypeScript Prisma ORM JWT
📬

Queue System

BullMQ on Redis for rate-limit-aware job scheduling. 4 worker types: ingestion, network crawl, signal scoring, on-chain lookup.

BullMQ Redis Bull Board Exponential backoff
🗄

Data Layer

Postgres for persistent relational data. Redis for hot cache and rate-limit counters. No raw tweet content stored (X ToS compliance).

PostgreSQL Redis Cache Prisma pg_trgm index
🔗

On-Chain Layer

Cross-references social signals with DexScreener (free), Birdeye, and Moralis for wallet/token activity. Async enrichment.

DexScreener API Birdeye Moralis Etherscan
☁

Infrastructure

Railway or Render for MVP. Worker pool scales horizontally. Upstash Redis for serverless-compatible queues. Migrate to AWS for scale.

Signal Detection Algorithm

4-layer scoring system — the core IP of CryptoRadar. Each layer adds a dimension of signal quality.

1

Account Quality Score (AQS)

Score each account 0–100 based on follower ratio, engagement rate, crypto-nativity

▾
// AQS = weighted sum of 5 factors (0–100) AQS = ( follower_to_following_ratio × 0.25 + engagement_rate × 0.30 // likes+replies / impressions + crypto_content_pct × 0.20 // % of tweets are crypto + account_age_score × 0.10 // older = more trusted + known_alpha_network × 0.15 // follows known alpha accs? )
Engagement Rate
30% weight
Highest weight — low follower + high engagement = strong signal
Follower Ratio
25% weight
Organic accounts have reasonable F/F ratios
Crypto Content %
20% weight
Measured via cashtag & keyword frequency
Network Overlap
15% weight
Do they follow other verified alpha accounts?
Account Age
10% weight
Older accounts less likely to be shills
2

Project Mention Scoring

Aggregate all mentions of a project, weighted by the AQS of mentioning accounts

▾
// Final project signal score (0–100) score = ( log(uniqueAccounts + 1) × 20 // reach + avgAQS × 0.4 // quality + min(velocity × 10, 25) // acceleration + crossNetworkScore × 0.15 // 2nd-degree reach ) → capped at 100
3

Early Signal Flags

Boolean boosters applied when specific pre-viral patterns are detected

▾
🚨 Pre-Viral
3+ high-AQS accounts mention it before any >10k follower account does
⚡ Velocity Spike
Mention rate doubled in the last 6 hours vs prior 6-hour window
🎯 Influencer Touch
Known alpha account liked/replied without posting yet — stealth signal
🔗 On-Chain Match
DEX listing <72h old with matching ticker/name found
🌱 New Account
Project's own X account is <30 days old with fast-growing engagement
🕸 Convergence
Accounts that don't normally interact are all talking about the same project
4

On-Chain Correlation

Boost score when social signal timing aligns with on-chain activity

▾
// Apply multiplier if on-chain data confirms signal if (dexListingAge < 72h) score *= 1.30 // very new listing if (liquidityGrowth > 200%) score *= 1.15 // LP being added if (freshDeployerWallet) score *= 1.10 // new deployer wallet if (noAudit) score *= 0.85 // risk penalty
